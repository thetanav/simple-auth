Dual-token pattern

Access token

Short-lived JWT (15 min) returned in JSON. Client sends it as Authorization: Bearer on protected routes. Payload: userId, sessionId, type: "access".

Refresh token

Long-lived JWT (7 days) in an httpOnly cookie. Only a SHA-256 hash is stored in Postgres. Used to mint new access tokens without re-login.

Refresh Token Flow

POST /auth/refresh-token — uses the httpOnly cookie to mint a new access token and rotate the refresh token in the database.

Prerequisite

Sign up or sign in first — that sets the refreshToken httpOnly cookie. This request sends it automatically via credentials: "include".

Logout Flow

POST /auth/logout — invalidates the current session server-side, deletes the refresh token from the database, and clears the httpOnly cookie.

What gets destroyed

Only the current session tied to your refresh cookie — not the user account. Other devices stay signed in.
